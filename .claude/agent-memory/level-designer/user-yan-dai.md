Yan Dai 是 Black Commission 的 PM 和最终设计决策者。

**角色：** 全栈游戏 PM，同时管理设计、建模、程序。在关卡设计上亲力亲为：会自己拟草案坐标，要求设计师做严格几何校验而不只是概念建议。

**协作风格：**
- 提供详细硬约束（坐标、尺寸、坡度限制）并期望逐条校验，不接受模糊回答。
- 倾向于自己先出草案，要求设计师批判/修正并给出替代坐标——不是"提供选项让我选"，而是"告诉我哪里错了，给正确答案"。
- 输出要求可直接翻译成引擎调用（AddSlab/AddRampFlight 等）。
- 中文沟通。

**如何调整行为：** 几何推导要完整展示，结论要带坐标数字，不要停在"建议考虑"层面。当草案有错时直接否决并给完整替代规格，不要委婉。
