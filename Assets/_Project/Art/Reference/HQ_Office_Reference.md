# 事务所 (HQ Office) 美术参考

> 把那张平面图 PNG 放到这个文件夹,命名为 `HQ_Office_Reference.png`,
> 这样我每次都能对照。下面这份文字稿是从那张图提炼的,即使没有 PNG 我也能参考。

## 整体尺寸
- 约 **18m (宽) × 14m (深)**,层高 **3m**(可落地到 Unity)
- 实际玩法落地的脚本坐标里,室内可走区约 X:[-5, 5.2]、Z:[-3.2, 3.2],车库门(车辆出口)在南侧 (−Z)

## 分区布局
- **左侧 = 办公室区**:委托电脑桌(核心交互)、收据打印机、债务公告板 (DEBT)、公司标识 (BLACK COMMISSION)、文件柜、二手沙发+茶几
- **右侧 = 车库区**:停着 BLACK COMMISSION 面包车、工具架/维修架、补给箱/物资箱、灭火器/急救箱
- **动线**:出生点(下方中央)→ 绿色 COMPUTER + 黄色箭头引导 → 接委托 → 走向面包车 → 卷帘门出发
- **出入口**:主入口(人行门,通停车场/街道,下方);卷帘门(车辆出入口,右下)

## 编号物件清单 (图中 1–12)
1. 委托电脑桌(核心交互) 2. 收据打印机 3. COMPUTER 标识(绿色灯箱)
4. 债务公告板 (DEBT 公告/账单/催款信) 5. 公司标识 (BLACK COMMISSION)

9. 补给箱/物资箱 10. 灭火器/急救箱 11. 地面导向线(黄漆) 12. 车库卷帘门

## 风格说明
- 低多边形,脏旧但可读
- 照明:办公室白炽暖光为主,保留少量阴影氛围,但保证可读性;绿色 CRT 终端作为标志性发光焦点;车库少量钠灯黄
- 所有交互/重要物件明确,玩家一眼能读懂目标与动线

## 落地状态(代码侧)
- 事务所由 [MvpSceneStyleDirector](../../Scripts/MVP/MvpSceneStyleDirector.cs) 程序化搭建
- `PreferBlenderOfficeFbx = false` → 用代码搭的可控外壳(本参考的目标),而非烤死的 FBX
- 家具:逐个用 Meshy 生成后接入,替换占位方块
