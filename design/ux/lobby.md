## Purpose & Player Need

玩家（host 或刚连上的 client）想要的是「确认会合成功，然后开工」。
派工名单卡回答三个问题：**谁在队里、我的工装是什么颜色、朋友进来要什么码**。
看完即走——大厅不是房间，**HQ 本身才是大厅**；这张卡只是进门前在门房签到的一拍。

PM 锁定（2026-06-12）：**各自进入**。每人看完名单自己确认到岗、各自推镜进
HQ；先进的先逛办公室，后来的队友随时加入。无空转等待，无房主门控。

## Player Context on Arrival

- Host：刚在 CRT 上选了「继续营业 / 新事务所」，主持成功的瞬间。情绪：期待开工。
- Client：刚提交加入码、进度墨条走完连上的瞬间。情绪：会合的轻快。
- 两者都无时间压力；这是全游戏压力最低的屏幕之一——也是唯一适合摆弄
  角色色板的地方。

## Navigation Position

`启动 → 主菜单（CRT）→ 大厅（派工名单卡）→ HQ`。
主菜单的模态子页之一（main-menu.md 子页表第 2 行），盖章公文卡语法。
**进入 HQ 后不可重开**——人员状态看世界本身 + 顶部小纸签（码+人数）。

## Entry & Exit Points

| 入口 | 触发 | 携带上下文 |
|---|---|---|
| 主持成功 | 继续营业/新事务所 → StartHost OK | 房间码（Relay）或 端口（LAN 直连） |
| 加入成功 | 调岗申请单 → 连接成功 | 会话花名册（自己已 spawn） |

| 出口 | 触发 | 备注 |
|---|---|---|
| 进入 HQ | Enter / [确认到岗] | 卡滑出 → 推镜交棒（~0.8s 无黑屏）→ 顶部小纸签出现 |
| 解散事务所（host） | Esc → 解散确认小卡 | 不可逆：断开全部队友，回主菜单 |
| 离开会话（client） | Esc → 离岗确认小卡 | 回主菜单 |
| 被除名（client，被动） | host 按住除名完成 | 本卡变「除名单」态 → 回主菜单 |
| 连接断开（被动） | 网络错误 | 卡上盖「失效」章 + [返回] |

## Layout Specification

### 载体

屏幕中央「盖章公文卡」（全游戏统一弹窗语法）：做旧纸底 #D6CCAE、市政青页眉
#3F5F5C、墨字、印章红 #C23A2B。出现 = 纸张 0.25s 滑入 + 落章音。卡片弹出时
背后 CRT/场景调暗 60%。字阶对齐结算单基准（卡头 14px / 章字 18px）。

**退役项（实施时）**：MainMenuUI 的 940×620 绿色「WaitingTerminal」面板 +
QuickNetworkUI 的 IMGUI HostWaiting 覆盖层 + 连接前旧面板上的色板选择
（DrawCharacterSelector / MainMenuUI 旧选择器）——全部由本卡取代。

### Information Hierarchy

1. **房间码**（页眉右侧，琥珀大字）——host 最常被队友问的一件事
2. **花名册 4 行**（卡体）：谁在、什么颜色、谁是负责人
3. **自己的色板选择**（嵌在自己那行）
4. **确认到岗**（卡底主行为）
5. host 行操作（静音/除名）——低频，弱化呈现

### Layout Zones

- **页眉带**（市政青）：左「派 工 名 单」+ 单位行小字；右房间码
- **卡体**：花名册 4 行（工号 01-04 固定槽位）
- **卡底**：[Enter] 确认到岗 + 一行墨淡提示
- **角落**：成立章（host 视角「事务所已开门」红章，装饰性）

### Component Inventory

| 组件 | 类型 | 交互 | 模式 |
|---|---|---|---|
| 房间码 | 琥珀大字（LAN 模式=「仅局域网 :端口」） | 只读 | — |
| 花名册行 ×4 | 工号+马甲色条+名字+职务章 | 自己行可改色板 | **花名册行（新模式，入库种子 #9）** |
| 色板选择 | 自己行内 ←→ 切换（快捷路径） | 撞色独占 | 同上 |
| 「更换工装」入口 | 自己行小字钮 | 切入火星监控室选人场景 | **监控室选人（新模式，PM 2026-06-12）** |
| 静音钮 | 他人行尾小字钮 | 点击切换 | — |
| 除名钮 | 他人行尾小字钮（仅 host） | **按住 0.8s** | 按住=签字（既有模式复用） |
| 确认到岗 | 卡底主行为行 | Enter / 点击 | 盖章公文卡确认 |
| 解散/离岗确认 | 二级小卡 | Enter 确认 / Esc 撤回 | 盖章公文卡 |

## States & Variants

| State | 触发 | 变化 |
|---|---|---|
| HostWaiting | 主持成功 | host 视角：他人行有 [静音][除名]；角落「成立」章 |
| ClientJoined | 加入成功 | 无 host 操作；负责人行有「负责」青章 |
| LAN 直连 | 无 Relay 码 | 码位显示「仅局域网 127.0.0.1:7778」墨小字 |
| 撞色 | 目标色已被占 | 色点变 ◌ 置灰 + tooltip「已登记」；切换跳过它 |
| 除名进行中 | host 按住除名 | 该行墨条 0→100% 填充，松手回流（0.3s） |
| 被除名（client） | host 完成按住 | 卡变「除 名 单」：红章「除名」+ 一行事由 + [返回] |
| 失效 | 连接断开 | 卡上盖「失效」红章 + 错误文案 + [返回] |
| 满员 | 第 5 人尝试加入 | 加入端在调岗申请单上收「驳回」章（main-menu spec 既有态） |

## Interaction Map

输入：键鼠（PC）。全链一词一义延续：**Enter=确认到岗、Esc=撤回/离开（带确认）、
按住=签字（除名）、←→=切色板、↑↓=行焦点（键盘流）**。

| 动作 | 输入 | 反馈 | 结果 |
|---|---|---|---|
| 切色板（自己行） | ←→ / 点击色点 | 色条+色点即时换色，哔声 | CharacterIndex 热改（全端同步） |
| 确认到岗 | Enter / 点击 | 落章音 + 卡滑出 | 推镜进 HQ，小纸签出现 |
| 静音队友 | 点击 [静音] | 字色 绿↔印章红 | 本地静音切换 |
| 除名（host） | **按住 [除名] 0.8s** | 行内墨条填充→落红章 | DisconnectClient；被踢端见除名单 |
| 离开/解散 | Esc | 弹二级确认小卡 | 确认后断开回主菜单 |
| 键盘行操作 | ↑↓ 选行 → Tab 在行内钮间移焦 → Enter/按住 Enter | 反白 | 同鼠标路径 |

## Events Fired

| 动作 | 事件 | 备注 |
|---|---|---|
| 切色板 | CharacterIndex NV 写（ServerRpc 校验独占） | **持久**：PlayerCharacterPalette.SavedIndex 同步落 PlayerPrefs |
| 确认到岗 | 本地卡收起 + 推镜；无网络事件 | 人早已 spawn，纯视角交棒 |
| 静音 | ProximityVoiceChat.SetClientMuted（本地） | 不同步给他人 |
| 除名 | NetworkManager.DisconnectClient(reason) | **架构注意**：disconnect reason 需携带「除名」标记供被踢端显示除名单 |
| 解散（host） | NetworkManager.Shutdown | 全员回主菜单 |

## Transitions & Animations

- **进卡**：纸张 0.25s 滑入 + 落章音（盖章公文卡统一拍）
- **出卡（确认到岗）**：纸张滑出 → 菜单相机 ~0.8s 推向玩家视角交棒，无黑屏
- **除名按住**：行内青墨条匀速填充 0.8s，松手 0.3s 回流（与提前收工申请单同手感）
- **除名单/失效**：红章 1.3→1.0 / 0.12s 砸落（全游戏统一章动效）
- 减少动效选项：延后（无设置系统，与 boarding-transit 残留项同归设置屏）

## Data Requirements

| 数据 | 来源系统 | 读/写 | 备注 |
|---|---|---|---|
| 花名册（名字/色板/host） | PlayerController DisplayName + CharacterIndex NV | 读 | 0.4s 轮询（现 RefreshLobbyWaiting 节奏可沿用） |
| 自己色板 | CharacterIndex NV | 写 | **需新增 ServerRpc 独占校验**（server-authoritative） |
| 入会色分配 | host 端 | 写 | 新人 join 时若偏好色被占，host 分配首个空闲色 |
| 静音状态 | ProximityVoiceChat | 读+写 | 本地 |
| 房间码/端口 | ConnectionManager / UnityTransport | 读 | — |
| 踢人 | NGO DisconnectClient | 写（host） | reason 需可区分「除名」vs 普通断线 |

**架构旗标（实施前需确认）**：①色板独占 = 新的 server-authoritative 小逻辑
（改色 ServerRpc + join 时分配），归 laplace/网络域；②NGO disconnect reason
携带除名标记（被踢端 UX 依赖它）。

## Accessibility

- 全键盘可达：↑↓ 行焦点、Tab 行内钮、←→ 色板、Enter 确认、Esc 返回
- 色板独占不靠纯色：被占色点 ◌ 形态变化 + tooltip 文字双通道
- 花名册行身份不靠纯色：负责人有「负责」文字章，不只看色条
- 纸底墨字对比度 ≥ 4.5:1（#D6CCAE 上 #19130D 级墨色，同主菜单 spec）
- 按住除名有可见进度（墨条），非纯计时
- 无障碍层级未定——沿用各 spec 的 WCAG-AA 基线建议（见 Open Questions）

## Localization Considerations

- 名字行：CJK/拉丁混排，超 12 字截断加省略号（行宽固定）
- 职务章「负责」固定二字（en: HOST）；章框定宽，HIGH PRIORITY 检查 en/de 溢出
- 「（空缺）」「已登记」「确认到岗」均短词，风险低
- 房间码/端口为 ASCII，无风险

## Acceptance Criteria

- [ ] 主持/加入成功后 1s 内派工名单卡出现，房间码清晰可读（LAN 模式显示端口）
- [ ] 1-4 人花名册 ≤0.5s 内反映加入/离开/改色（名字、色条、负责章正确）
- [ ] 撞色独占：他人已占颜色不可选中；两端同时抢同色时 server 仲裁，败方回滚到原色且 UI 无残留错色
- [ ] host 按住除名 0.8s：中途松手取消无副作用；完成后被踢端显示「除名单」卡并返回主菜单
- [ ] Enter 确认到岗 → 推镜进 HQ 全程无黑屏；顶部小纸签常驻显示房间码+在线人数
- [ ] 连接断开：卡上「失效」章 + 错误文案 + [返回] 可用（client 端）
- [ ] 全部交互纯键盘可走通（↑↓/Tab/←→/Enter/Esc）
- [ ] 1280×720 下卡片不裁切、文字可读

## 修订：火星监控室选人场景（PM 2026-06-12，名单卡入口制）

参考意象：暗暖室内 + 亮窗带 + 雾中逐个走过的背包客剪影（PM 提供山屋参考图）。
- **入口**：派工名单卡自己行「更换工装」→ 切入监控室场景（不打断「各自进入」
  主流程；‹ › 行内快速换色保留为快捷路径）。
- **场景**：暗色监控小隔间（运行时生成），亮窗带，候选人穿各色工装在雾尘中
  逐个走过；←→/等待 = 下一位走过，Enter = 盖章选定 → 回名单卡。
- **撞色独占沿用**：已被占的工装走过时带「已登记」标记，不可选定。
- **叙事框架待 writer pass**：事务所在地球、火星是买家——倾向做成「劳务派遣
  资格审查窗口」（你挑人，也被挑），避免字面「我们在火星」的世界观矛盾。
- 实施估计 ~1 个工作日（隔间 + 走步小人 + 雾板 + 选定落章）；排在主菜单
  批次 1 编译验证之后。
