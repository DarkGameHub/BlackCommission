## Purpose & Player Need

玩家到达主菜单时想要的是「开工 / 和朋友会合」。设计目标：30 秒内完成
继续营业（单人/主持）或 加入事务所（输码进队友的局）。菜单同时承担
首次印象职责：建立 Municipal Debt Noir + lo-fi 工业恐怖的统一视觉身份。

## Player Context on Arrival

- 启动游戏后的第一屏；也可能是从一局游戏退出后返回。
- 情绪假设：平静/期待，无时间压力。自愿到达。

## Navigation Position

根屏幕。`启动 → 主菜单`。所有子页（加入/大厅/设置/直连/退出确认）都是
主菜单之上的模态卡片，Esc 逐级返回主菜单。

## Entry & Exit Points

| 入口 | 触发 | 携带上下文 |
|---|---|---|
| 游戏启动 | 自动 | 无（读取存档判断「继续营业」可用性） |
| 局内退出 | 断开会话 | 上一局的存档状态 |

| 出口 | 触发 | 备注 |
|---|---|---|
| 进入 HQ（host） | 继续营业 / 新事务所 → 大厅 → 进入 | 相机推镜交棒，无黑屏 |
| 进入 HQ（client） | 加入事务所 → 连接成功 → 大厅 → 进入 | 同上 |
| 退出游戏 | CRT 第 5 行「关机」→ 离岗单确认 | 两步确认，不可逆 |

## Layout Specification

### 载体（核心决策，PM 锁定）

**实时 3D 办公室**：菜单相机固定架在真实 HQ 场景内，朝向办公桌 CRT。
删除写实烘焙 PNG（MainMenuBg/MenuPanel 退役）。lo-fi 渲染 + LC 描边 +
后处理对菜单全部生效——菜单即游戏画面本身。

构图约束（具体机位进编辑器调）：
- CRT 显示器居左 1/3，轻微俯角（坐到桌前的视角）
- 窗外/门外货车剪影入画右侧
- 墙上催缴单（市政青+印章红）入画
- 台灯钨丝暖光为主光源；CRT 绿光为第二光源

### Information Hierarchy

1. **四个主行为**（CRT 屏上终端绿行）：`> 继续营业 / 新事务所 / 加入事务所 / 设置`
   + 第 5 行 `关机`（=退出游戏，flavor 化）
2. **新委托招贴**：桌面上钉着的纸条（当前可接任务标题+报酬，真实任务数据驱动）
3. 氛围层（货车、催缴单、台灯）+ 版本号（CRT 屏角磷光小字）

### Layout Zones

- **CRT 屏幕区**（左 1/3）：World-space Canvas 贴在屏幕表面，终端绿菜单行
- **桌面区**（下沿）：委托招贴纸条
- **场景纵深区**（右 2/3）：氛围叙事，不可交互
- **模态层**（屏幕中央）：纸质表单卡片（子页）

### Component Inventory

| 组件 | 类型 | 交互 | 模式 |
|---|---|---|---|
| 菜单行 ×5 | 终端文字行 | hover 反白+哔声 / 点击 | CRT 终端行（新模式） |
| 委托招贴 | 纸条+章 | 只读（数据驱动） | 公文纸（与塔楼封条同语法） |
| 子页卡片 ×7 | 模态纸卡 | 表单交互 | 盖章公文卡（全游戏统一弹窗语法） |
| 版本号 | 磷光小字 | 无 | — |

### 子页语法（PM 锁定：纸质表单卡片）

屏幕中央弹出「盖章公文卡」：做旧纸底 #D6CCAE、市政青页眉 #3F5F5C、
墨字、印章红 #C23A2B 确认章、`[确认][返回]`。出现 = 纸张滑入 + 落章音
（StampThunk）。卡片弹出时背后 CRT/场景调暗 60%。

| 子页 | 公文身份 |
|---|---|
| 加入事务所 | 调岗申请单（代码输入框） |
| 大厅等待 | 派工名单（1-4 花名册行 + 角色色板选择） |
| 设置 | 偏好登记表 |
| 直连 LAN | 申请单变体（IP/端口） |
| 退出确认 | 离岗单（两步确认） |
| 连接中 | 纸卡 + 青色进度墨条 |
| 已连接状态 | 顶部常驻小纸签 |

## States & Variants

| State | 触发 | 变化 |
|---|---|---|
| Main | 默认 | CRT 五行可选 |
| JoinInput | 加入事务所 | 调岗申请单卡 |
| Connecting | 提交代码/直连 | 进度墨条卡，可取消 |
| HostWaiting | 主持后 | 派工名单卡（实时花名册） |
| Settings / DirectConnect / QuitConfirm | 对应行 | 对应公文卡 |
| ConnectedStatus | 已在会话中 | 顶部小纸签常驻 |
| 错误/驳回 | 连接失败/码错 | 当前卡上盖「驳回」红章 + 错误文案 |
| 无存档 | 首次启动 | 「继续营业」行灰字（墨淡）不可选 |

## Interaction Map

输入：键鼠（PC）。键盘流：↑↓ 选行、Enter 确认、Esc 逐级返回；
全部子页可纯键盘走通。鼠标：射线命中 world-space canvas 行。

| 动作 | 输入 | 反馈 | 结果 |
|---|---|---|---|
| 选行 | ↑↓ / hover | 反白绿条 + 哔声 | 高亮 |
| 确认 | Enter / 点击 | 落章音（子页）/ 哔声 | 进入子页或开局 |
| 返回 | Esc / [返回] | 纸卡滑出 | 上一级 |
| 输码提交 | Enter | 进度墨条 | Connecting |

## Events Fired

| 动作 | 事件 | 备注 |
|---|---|---|
| 继续营业/新事务所 | StartHost + 存档读写 | 持久状态，架构已有 |
| 加入提交 | StartClient(code) | — |
| 设置修改 | PlayerPrefs 写入 | 即时生效 |
| 关机确认 | Application.Quit | 不可逆 |

## Transitions & Animations

- **进入菜单**：CRT 开机闪（ComputerBoot 音已有）
- **进入游戏**：相机从桌前推向玩家出生点（~0.8s）后交棒玩家相机——
  菜单到游戏无黑屏（直接回应「每一步像单独板块」）
- **子页**：纸卡滑入/滑出 + 落章；连接中 = 墨条填充
- 减少动效选项：未定（无障碍层级未定，见 Open Questions）

## Data Requirements

| 数据 | 来源系统 | 读/写 |
|---|---|---|
| 存档存在性/公司状态 | CompanyData/SaveIO | 读 |
| 可接任务（招贴条） | Resources/Tasks 任务资产 | 读 |

| 设置项 | PlayerPrefs | 读+写 |

## Accessibility

- 全键盘可达（上述键盘流）
- 终端绿行 hover 反白（非纯色依赖：反白+前缀 `>` 双通道）
- 纸卡墨字对比度 ≥ 4.5:1（做旧纸底 #D6CCAE 上用 #19130D 级墨色）
- 无障碍层级未定 — 见 Open Questions

## Localization Considerations

- 菜单行中英混排（现状），行宽 340px 上限——德/法文 +40% 会爆，
  HIGH PRIORITY：行布局需弹性宽度
- 报酬/代码为数字，无本地化格式风险

## Acceptance Criteria

- [ ] 启动到菜单可交互 < 3s
- [ ] 四主行为 + 全部 7 个子页键盘可达（↑↓/Enter/Esc 走通）
- [ ] 加入码输错 → 当前卡显示「驳回」章态 + 文案
- [ ] 大厅正确显示 1-4 人花名册（名字/角色色板/host 标识）
- [ ] 1280×720 下全部文字可读、纸卡不裁切
- [ ] 招贴条内容来自真实任务资产（换任务资产文案自动变）
- [ ] 进入游戏的推镜转场无黑屏、无相机跳切
- [ ] 「关机」需经离岗单两步确认
