## Purpose & Player Need

玩家走到办公电脑前想做的是三件事之一：**领上一单的钱 → 接下一单 → 补给装备**。
这是游戏循环的起点和终点闭合处——经济系统的主表面（office-economy-progression GDD
定位），也是 hud.md 哲学里唯一的「大音量信息面」：全游戏其它界面刻意低调，只有这块
CRT 允许信息密集。

设计目标：归来玩家 30 秒内完成「领结算→扫一眼新委托→接单」；备战玩家 60 秒内完成
「查账→采购→出门上车」。

---

## Player Context on Arrival

两种典型到达时刻：

- **任务归来**（带 pending reward）：情绪是紧张释放+盘点，第一眼想看「这单赚了多少、
  现在还欠多少」。结算讽刺文本（客户使用备注）在此时被阅读。
- **新一轮开工**：情绪平静、计划性，想挑活和算账。
- **首次接触** = 第一局开局教学；教学收购（150G 收购文件）也在这屏发生。

全部自愿到达（走近+E），无强制弹出。多人时 1-4 人可能同时围观，但只有 host 能执行
经济操作（client 看到「等房主」态）。

---

## Navigation Position

Diegetic 屏幕，不在任何菜单层级里：`HQ 自由行走 → 走近办公桌 CRT → E 进入特写`。
进入 = 相机推近到 CRT 正面（`ComputerCloseupCamera` 已有此机制），UI 是贴在 CRT
屏幕上的 world-space canvas——与主菜单 spec 同载体。Esc 或后退退出特写回到行走。

**PM 锁定（2026-06-11）：主菜单 CRT 与办公电脑是同一台机器。** 主菜单态显示
「BC-DOS 启动菜单」（继续营业/新事务所…），入局后同一块屏切换为「办公管理系统」。
主菜单推镜的终点 = 玩家出生点，路过的就是这台机器——世界连续性最强，复用同一块
world-space 屏幕载体。

## Entry & Exit Points

| 入口 | 触发 | 携带上下文 |
|---|---|---|
| HQ 行走中 | 走近 CRT + E | 交互提示随状态变：「领取结算」/「查看已锁定委托」/「事务所收购文件」/「打开委托终端」（现有 `InteractHint` 逻辑保留） |
| 主菜单推镜交棒 | 进入游戏 | 不自动打开终端，玩家自己走过去（保持自愿性） |

| 出口 | 触发 | 备注 |
|---|---|---|
| 回到行走 | Esc / S 后退 | 无状态变化 |
| 接单成功 | 「接受委托」确认 | 终端自动合上 + 提示去上车（单向：接单后委托锁定，不可在终端撤销） |
| 走远 | 距离超限 | 防多人卡死：别人也要用电脑 |

## Layout Specification

### Information Hierarchy

**有待领结算时**（归来态）：
1. **待结算文件块**——金额+结果（领取前商店锁定，最显眼）
2. 资金/债务（领完立刻看余额变化）
3. 其余一切

**默认态**（开工态）：
1. **可接委托**（标题/委托人/地点/报酬/时长）——循环起点
2. **资金 + 执照阶段**——常驻状态条（GDD 硬性要求；负数=警示态）
3. 采购目录（F1–F4）
4. 已锁定委托状态 / 教学收购 offer（条件触发时插队到 #1）
5. host/client 权限态（client 全程只读+「等房主」）

**永不显示**（GDD 禁令）：声望、办公室等级、收购压力数值。

### CRT 配色（PM 锁定 2026-06-11）

**单色绿磷光屏**：警示/负数 = 反白行（绿底黑字）+ 前缀 `!`，不引入红色。
印章红只出现在纸卡模态（屏幕空间），不上 CRT——符合 art bible 用色禁令，
且天然满足非纯色依赖的无障碍要求。

### Layout Zones（页签式终端，PM 锁定 2026-06-11）

| 区带 | 内容 | 常驻性 |
|---|---|---|
| **Z1 顶栏** | `BC 办公管理系统 v2.1` + 日期 2098-XX-XX + `USER: BC_STAFF` | 常驻 |
| **Z2 状态条** | 资金（负数反白）｜债务｜执照阶段｜连接态（HOST / 等房主） | 常驻 |
| **Z3 页签行** | `[1] 委托文件` `[2] 采购目录` `[3] 公司账本`，数字键直达 | 常驻 |
| **Z4 内容区** | 随页签切换 | — |
| **Z5 行动栏** | 单一主行动 `› [E] …`（GDD 优先级：领结算→接单→确认收购）+ 按键提示 | 常驻 |

### 页签内容

**[1] 委托文件**（默认页签）
- 委托表 `编号｜委托名称｜委托人｜报酬｜状态`——**全部数据驱动**，无硬编码假行：
  可接（真实任务池）+ 已归档（完成单灰行留痕，越玩越厚）+ 特殊文件
  （教学收购条件满足时「事务所收购文件 150G」高亮特殊行插入）
- 选中行下方 = 详情块：地点/报酬/预计时长/结局方式（全部/部分/失败）/客户备注
- **待领结算时**：表格顶部插入反白「待结算文件」块——结果+金额一行+
  「领取后解锁采购」，E=领取；领取后变「已入账 +NG」闪 2s 后归档
- 空池：`本周期暂无新委托——等待派单`

**[2] 采购目录**
- 货品行 `F1 手电筒 30G｜F2 电池 15G｜…`（数据驱动，含持有数）；F1–F4 全局可用
- 资金不足按下 → 该行反白闪 + `! 资金不足`
- 待结算未领 → 整页蒙「采购冻结：先领取待结算文件」

**[3] 公司账本**
- 资金/债务当前值 + 最近 N 单结算流水（含**客户使用备注讽刺文本**——
  结算屏一次性，账本可回看）
- 不显示声望/等级/压力（GDD 禁令）

## States & Variants

| State | 触发 | 变化 |
|---|---|---|
| Default | 打开，无待结算无锁定 | Tab1，E=接受委托 |
| **待结算** | 任务归来未领 | 反白块置顶呼吸，E=领取（仅 host），采购冻结 |
| 入账闪现 | 领取成功 | 「已入账 +NG」反白闪 2s → 归档进账本 |
| **委托已锁定** | 已接单 | 委托行状态=已锁定，主行动变灰「委托已锁定——去外面上车」 |
| 收购 offer | 满足教学收购条件 | 特殊高亮行插入 + E=确认收购（优先级在领结算之下、接单之上，按 GDD） |
| **Client 视角** | 非 host | 状态条「等房主」，一切经济操作灰锁，只读浏览可以 |
| 空任务池 | 无可接任务 | `本周期暂无新委托——等待派单` |
| 资金不足 | F 键购买失败 | 该货品行反白闪 + `! 资金不足` |
| 离线/单机 | 无网络会话 | 连接态=离线；solo 模式允许接单（现有 allowNonNetworkSoloStart 逻辑） |
| 调度中 | 接单后货车调度 | 主行动「委托车调度中…」 |

## Interaction Map

输入：键鼠（PC）；world-space 射线点击与主菜单同机制。全部子流程纯键盘可走通。

| 动作 | 输入 | 反馈 | 结果 |
|---|---|---|---|
| 切页签 | 1/2/3 / 点击 | 哔声 + 即时重绘 | 切换内容区 |
| 选委托行 | ↑↓ / 点击 | 反白选中条 + 哔声 | 详情块更新 |
| 主行动 | E / Enter / 点击行动栏 | 哔声；成功=确认音 | 按优先级执行（领结算→接单→确认收购） |
| 购买 | F1–F4（任意页签） | 成功=哔+热键栏立即出现图标数量（GDD 要求）；失败=反白闪 | 资金扣减+发货 |
| 退出 | Esc / S | 相机拉回 | 回到行走 |

## Events Fired

全部走现有 host-authoritative 通道；无新增分析事件（项目无 analytics 管线）。

| 动作 | 事件 | 持久状态 |
|---|---|---|
| 接受委托 | `TryAcceptDemoTask` → 任务锁定 + 货车调度 | ✅ 任务运行时 |
| 领取结算 | `MvpPendingReward.Claim` → CompanyState 结算 + 存档 | ✅ 资金/存档 |
| 确认收购 | `TryAcquireTutorialOffice` | ✅ 资金/执照 |
| 购买 | 商店购买 RPC → 热键栏 + 扣款 | ✅ 资金/背包 |
| 切页签/选行 | 无事件（纯本地 UI） | — |

## Transitions & Animations

- **进入**：相机推近 CRT ~0.4s（复用 `ComputerCloseupCamera`）。屏幕是 world-space
  常驻渲染——没进特写时路过也能瞥见屏上内容（diegetic）
- **退出**：相机拉回 ~0.3s
- **页签切换**：无平滑动画——CRT 即时重绘 + 一帧扫描线闪（终端没有缓动）
- **待结算块**：反白 1Hz 呼吸；**入账**：反白闪 2s + 确认音
- 减少动效：CRT 闪烁/呼吸应可关——并入全局无障碍层级（见 Open Questions）

## Accessibility

- 纯键盘走通一切：1-3 / ↑↓ / E / Esc / F1–F4
- 单色屏天然非纯色依赖：警示=反白+`!` 前缀双通道
- 特写下文字 ≥ 等效 16px@720p；磷光绿/黑底对比度 >7:1
- CRT 闪烁/呼吸可关（光敏保护）
- 无障碍层级未定——沿用主菜单 spec 的 Open Question（建议 WCAG-AA 基线）

---

## Localization Considerations

- 页签/状态条中英混排，页签行宽需弹性（德法 +40% 会爆）——**HIGH PRIORITY**，
  与主菜单 spec 同一条
- 客户备注讽刺文本=长文本，详情块需自动换行
- 货币为 `NG` 数字，无本地化格式风险

---

## Acceptance Criteria

- [ ] E 打开到特写可交互 < 0.6s
- [ ] 终端无任何硬编码任务行；任务池为空显示「本周期暂无新委托」
- [ ] 状态条**不**显示声望/办公室等级/压力；显示资金（负数反白`!`）+执照+连接态
- [ ] 待结算未领时 F1–F4 全部被拒并显示「采购冻结」；领取后立即可购
- [ ] host 领取结算幂等（重复 E 无二次入账），资金/账本即时更新
- [ ] client 视角经济操作全部灰锁+「等房主」，浏览/切页可用
- [ ] 购买成功后热键栏立即出现图标+数量
- [ ] 纯键盘走通全部页签与操作
- [ ] 接单成功后终端自动合上，重新打开显示「委托已锁定」
- [ ] 1280×720 特写下全部文字可读
